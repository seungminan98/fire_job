# fire_job — 소방공사 일정 · M/M Capacity 관리 웹

소방공사 프로젝트의 **일정(간트)** 과 **월별 투입 인력(M/M, man-month) Capacity** 를
한 화면에서 관리하는 웹앱입니다.

> 설계 철학: **파일 하나 · DB 없음 · 무료.**
> 데이터는 평소 브라우저(localStorage)에 자동 저장되고, 공유는 URL/CSV 두 갈래로 안전장치를 뒀습니다.

---

## ✨ 한눈에 보기

- HTML 파일 **하나로 완결** — 외부 라이브러리·CDN·서버·DB 전부 없음
- 입력하면 **자동 저장**(localStorage), 새로고침해도 유지
- **공유 URL 복사** 한 번이면 서버 없이 데이터 전달(LZ-String 압축)
- 착수일만 넣으면 **7단계 일정 자동 분배**
- **간트차트 · Capacity/Pull · 인원별** 3개 미리보기
- 표 컬럼/정렬/뷰를 자유롭게 커스터마이징

---

## 🧱 기술 스택

| 구분 | 내용 |
|------|------|
| 마크업/스타일 | HTML5, CSS3 (CSS 변수 기반 디자인 토큰, Flexbox/Sticky) |
| 로직 | Vanilla JavaScript (프레임워크 없음, 빌드 과정 없음) |
| 차트 | 인라인 **SVG** 직접 렌더 (꺾은선 차트), HTML/CSS 기반 간트 막대 |
| 데이터 저장 | 브라우저 **localStorage** (자동 저장) |
| 공유/직렬화 | JSON → enum 인덱스화 → **LZ-String**(MIT, 인라인) 압축 → URL 해시 |
| 입출력 | CSV 내보내기/불러오기(직접 구현한 파서·직렬화) |
| 배포 | 정적 호스팅(GitHub Pages 등) 또는 파일 직접 열기 |

**의존성 0** — `index.html` 한 파일만 있으면 동작합니다. (LZ-String도 파일 안에 인라인)

## 🚀 사용 방법

### 방법 1. 파일 직접 열기 (가장 간단, 자동저장 O)
1. `index.html` 다운로드
2. 더블클릭해서 브라우저로 열기
3. 끝. 입력 내용은 그 브라우저에 자동 저장됩니다.

### 방법 2. GitHub Pages 등 정적 호스팅
1. 이 저장소를 GitHub Pages로 배포
2. 발급된 주소로 접속해서 사용 (자동저장 동작)

> ⚠️ 일부 미리보기 환경(샌드박스 등)에서는 localStorage 자동저장이 막혀 있을 수 있습니다.
> 그럴 땐 화면 상단에 **빨간 경고 배너**가 뜨며, **"전체 내보내기(백업)"** 또는 **"공유 URL 복사"** 로 보관하세요.

## 📖 사용설명서 (기능 상세)

### 1. 공사 목록 관리 (입력 표)
- 공사별 입력 항목: 공사명 · 규모(대/중/소) · 구분(경영계획/의뢰공사/자체공사) · 계약구분(입찰/단가) ·
  상태(예정/진행중/완료/hold/drop) · 연면적 · 층수 · **착수일 · 종료일** · 현단계 ·
  담당자(기계/전기/**정/부**) · 비고
- 공사 **추가 / 삭제**
- **행 순서 드래그 이동** (⠿ 핸들을 잡고 위아래로, 드롭 위치 미리보기 표시)
- **규모 색 구분**: 대 = 빨강 · 중 = 파랑 · 소 = 회색
- **구분 색 구분**: 경영계획 = 보라 · 자체공사 = 초록 · 의뢰공사 = 검정
- **상태별 행 강조**: 완료·hold·drop 행은 **회색 처리**, 완료·drop 행은 추가로 **취소선**

### 2. 단계 일정 자동 분배 (7단계)
- 단계: 접수 · 기획 · 설계 · 계약 · 시공 · 정산 · 인수인계
- **착수일만 넣으면 단계가 자동 분배** — 기획·설계·계약·정산·인수인계는 고정 기간,
  남은 기간을 접수 : 시공 비율로 분할
- 전체 기간이 고정 합보다 짧으면 고정분이 비례 축소
- **종료일 직접 편집 가능** — 종료일을 바꾸면 착수일~종료일 전체 기간에 맞춰 단계가 재분배됨
- **단계별 종료일 체인**: 한 단계 종료일을 바꾸면 다음 단계들이 자동으로 이어서 밀림
- **단가계약이면 '계약' → '내역검토'로 라벨 자동 전환**
- 단계 종료일은 **행별 펼치기(▸/▾)** 로 표시, 전체 펼치기/접기 버튼, 새 공사는 자동 펼침

### 3. 컬럼 커스터마이징
- 컬럼 **숨기기 / 보이기**
- 컬럼 **폭 조절** (헤더 우측 끝 드래그)
- 컬럼 **순서 이동** (헤더 드래그)
- **사용자 정의 컬럼 추가** (텍스트 / 숫자 / 선택형 드롭다운), 편집 · 삭제
- 종료일 컬럼은 기본적으로 착수일 바로 뒤에 배치됨

### 4. 정렬
- 헤더 클릭: 오름차순 → 내림차순 → 해제 순환
- **Shift+클릭으로 다중 정렬** (우선순위 1·2·3차, 헤더에 ▲¹ ▼² 표시)
- 타입별 똑똑한 비교: 규모는 대<중<소, 날짜는 날짜순, 숫자는 수치, **빈값은 항상 뒤로**

### 5. 뷰(프리셋)
- 컬럼 구성 · 순서 · 폭 · 숨김 · 정렬을 **뷰 단위로 저장**해 탭으로 전환
- 뷰 **추가 · 이름변경 · 삭제** (예: "착수일순 뷰", "규모·상태순 뷰"를 따로)

### 6. 미리보기 (3개 탭)
- **간트차트**: 각 단계를 **실제 기간만큼 이어지는 가로 막대**로 표현(색 = 단계).
  막대에 마우스를 올리면 기간·투입 M/M 툴팁 표시. **오늘 위치를 빨간 점선**으로 표시.
  좌측 표의 정렬·순서를 그대로 반영
- **Capacity / Pull**: 월별 **필요 M/M 합계 vs 가용 인원**, 가동률·판정(가능/빠듯/초과).
  상단에 **꺾은선 차트 + 가용 인원(budget) 점선**
- **인원별**:
  - **담당자 요약(피벗)**: 사람별 `정 공사 수 / 부 공사 수 / 담당 공사(중복 제외)`
  - **인원별 월 Capacity**: 사람별 월별 부담 M/M(색상 구분).
    상단에 **사람별 꺾은선 차트 + 1.0(1인 풀, budget) 점선**
  - 한 사람의 한 달 풀(full) 부담을 **1.0 M/M** 로 보고, **정 100% · 부 70%** 비중으로 합산(설정에서 조정)

### 7. M/M 설정
- 규모별 · 단계별 M/M 투입량 표
- 자동분배 기준(단계별 고정 개월수, 접수 : 시공 비율)
- 가용 인원 · 빠듯% · 초과% 임계값
- **정 비중 / 부 비중**(인원별 Capacity 합산 비율, 기본 100% / 70%)
- 기본값 복원

### 8. 저장 — localStorage 자동저장
- 입력할 때마다 브라우저(localStorage)에 자동 저장 → **새로고침해도 유지**
- 우상단에 "저장됨" 표시
- **저장 불가 환경이면 상단에 빨간 경고 배너** (새로고침 시 손실 위험 안내)

### 9. 공유 — JSON 압축 → URL (DB 없이)
- **"공유 URL 복사"** → 현재 데이터를 **배열 직렬화 + enum 인덱스화 + 단계 종료일을 일수 오프셋으로 저장 + LZ-String 압축**
  → URL 해시(`#d=...`)에 담아 클립보드로 복사
- 받은 사람이 URL을 열면 복원 제안 → 자기 localStorage에도 저장, 주소창의 긴 해시는 자동 정리
- URL이 길면 길이 경고
- **직접 수정한 단계 날짜가 그대로 보존됨**(예전 버전은 착수일로 재계산하느라 날짜가 틀어졌는데, 이를 해결)
- LZ-String은 MIT 라이선스로 HTML에 인라인 → 외부 의존 0

### 10. CSV 입출력
- **전체 내보내기(백업)**: 전체 컬럼 · 단계별 날짜 · 원본 순서 → 다시 불러오기 가능
- **현재 뷰 내보내기**: 화면 그대로(정렬 순서 · 보이는 컬럼 · 컬럼 순서) 보고/공유용
- CSV **불러오기** (기존 목록 대체 또는 추가 선택)
- **설정값**도 별도 CSV로 내보내기 / 불러오기
- 한글 깨짐 방지를 위해 UTF-8 BOM 포함

### 11. 레이아웃 · 배포
- 단일 컬럼 중앙 정렬, 표·간트는 폭에 맞게 펼쳐지고 넘치면 가로 스크롤
- **HTML 파일 하나로 완결** — 외부 라이브러리·CDN·서버·DB 전부 없음
- GitHub Pages 등 정적 호스팅에 그대로 올려 사용 가능

## 💾 데이터 보관 가이드

| 상황 | 권장 방법 |
|------|-----------|
| 평소 작업 | 자동 저장(localStorage) — 별도 조작 불필요 |
| 백업 / 다시 불러오기 | **전체 내보내기(백업) CSV** |
| 보고 · 출력용 | **현재 뷰 내보내기 CSV** |
| 다른 사람에게 빠르게 전달 | **공유 URL 복사** |
| 자동저장 불가 환경 | 작업 직후 백업 CSV 또는 공유 URL로 보관 |

---

## ⚠️ 주의사항
- localStorage는 **브라우저·기기마다 별도**입니다. 다른 PC에서 이어 작업하려면 CSV/URL로 옮기세요.
- 브라우저 데이터(캐시/사이트 데이터)를 지우면 저장 내용도 사라집니다 → 중요 데이터는 CSV 백업 권장.
- 공유 URL은 길어질 수 있어 일부 메신저에서 잘릴 수 있습니다. 길이가 길면 CSV도 함께 보관하세요.
